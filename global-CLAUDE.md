# 全局规则

<!-- 这个文件放在 ~/.claude/CLAUDE.md，对你所有项目生效。

## 回复偏好

- 始终用中文回复
- 回答要简洁直接
- 不要使用 emoji
- 不要在每次操作后总结你做了什么，我能看到
- 不要创建 README 或文档文件，除非我明确要求

## 通用铁律

<!-- 铁律 = 所有项目中绝对不能违反的规则 -->

### 铁律：敏感信息保护
- 绝对不要提交 .env、API Key、密码、token 到 git
- Push 前必须扫描代码中的硬编码密钥
- 发现泄露立即警告，不要继续 push

### 铁律：禁止未经确认的破坏性操作
- 未经我确认，不要执行：rm -rf、force push、reset --hard、checkout .
- 不要删除分支、数据库表、或任何用户数据
- 不要在没有备份的情况下覆盖重要文件

### 铁律：不要擅自公开仓库
- 绝对不要把任何 GitHub 仓库设为 public，除非我明确要求
- 创建新仓库时默认 private

## Git 安全

- 不要使用 `git push --force`（用 `--force-with-lease` 如果真的需要）
- 不要 `git commit --amend` 已经 push 过的 commit
- 不要跳过 pre-commit hook（`--no-verify`）
- 不要在 commit message 中包含敏感信息

## 依赖安全

- 不要自作主张添加新的依赖包，先问我
- 安装依赖后检查是否有其他包被意外移除
- 不要使用明显过时或已废弃的包

## 通用工作习惯

- 修改代码前先读代码，理解上下文
- 保持简单，不要过度设计
- 只做我要求的事情，不要额外添加功能或重构
- 改完代码先跑 lint 和测试，确认没有破坏现有功能
- 提到日期、时间前先运行 `date` 确认
- 遇到不确定的情况，问我而不是猜测
