每日开发的启动仪式。帮我快速回顾昨天的工作并聚焦今天的重点。

步骤：

1. **昨天做了什么** — 运行 `git log --since="yesterday" --oneline` 列出昨天的提交
2. **未完成的工作** — 搜索代码中的 TODO/FIXME 注释，列出待办事项
3. **当前分支状态** — 运行 `git status`，检查：
   - 有没有未提交的改动（昨天忘了 commit？）
   - 有没有未推送的 commit
   - 当前在哪个分支（是不是还在 feature 分支上？）
4. **CLAUDE.md 检查** — 快速看一下 CLAUDE.md 的决策日志和已知陷阱，提醒我最近记录的重要事项

输出格式：

```
## 每日摘要

### 昨天完成
- [commit 列表]

### 未完成
- [TODO 列表或"无"]

### 当前状态
- 分支：xxx
- 未提交改动：N 个文件
- 未推送 commit：N 个

### 提醒
- [从 CLAUDE.md 中提取的最近 1-2 条重要注意事项，或"无"]
```

规则：
- 保持简洁，不要超过 20 行
- 如果一切干净（无 TODO、无未提交），直接说「一切就绪，可以开始」
