---
description: "规划一个功能或任务：拆解步骤、评估风险、输出可执行的计划"
---

帮我规划一个功能或任务。按照以下流程进行：

1. **理解需求** — 如果我没有说清楚，用一个问题确认核心目标
2. **分析现状** — 快速浏览相关代码和文档，了解当前项目状态
3. **拆解步骤** — 将任务分解为 3-7 个可独立完成的步骤，每步标注：
   - 预计改动的文件
   - 风险等级（低/中/高）
   - 依赖关系（是否需要先完成其他步骤）
4. **输出计划** — 格式如下：

```
## 计划：[任务名称]

### 步骤
1. [步骤描述] — 改动：[文件] — 风险：低
2. [步骤描述] — 改动：[文件] — 风险：中
   - 注意：[潜在风险说明]
3. ...

### 注意事项
- [需要特别注意的坑]

### 验证方式
- [怎么确认任务完成]
```

规则：
- 不要写代码，只输出计划
- 每个步骤要足够具体，能直接照着做
- 如果任务太大（超过 7 步），建议拆成多个子任务
