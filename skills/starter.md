# 项目脚手架

帮用户从零开始搭建一个新项目，从想法到能跑的代码。

## 步骤

### 1. 了解需求

依次提问（每次一个）：

**问题 1：你想做什么？**
> 用一两句话描述你想做的东西。比如"一个个人博客"、"一个待办事项应用"、"一个 API 服务"。

**问题 2：有技术栈偏好吗？**
> 你想用什么技术？比如 Python、React、Next.js。
> 如果不确定，我根据你的需求推荐。

根据用户回答，推荐合适的技术栈。推荐时给出理由：
> 根据你的需求，推荐用 xxx，原因是 xxx。可以吗？

### 2. 搭建项目

用户确认技术栈后，按以下顺序搭建：

1. **初始化项目** — 使用官方脚手架命令（如 create-next-app、npm init、uv init 等）
2. **配置基础结构** — 创建源代码目录、测试目录、配置文件
3. **安装核心依赖** — 只安装实现核心功能必需的依赖
4. **创建入口文件** — 写一个最小可运行的示例代码
5. **配置 CLAUDE.md** — 自动运行 /setup 填写项目信息

### 3. 验证可运行

运行项目，确认能正常启动：
- 前端项目：启动开发服务器，确认页面可以打开
- 后端项目：启动服务，确认 API 可以响应
- CLI 工具：运行示例命令，确认输出正确

### 4. 告诉用户下一步

> 项目搭建完成，已经可以运行了。
>
> 项目结构：
> （展示关键目录和文件的简要说明）
>
> 接下来你可以：
> - 用 /auth 添加登录注册功能
> - 用 /database 连接数据库
> - 用 /api 生成后端接口
> - 用 /landing 做一个落地页
> - 用 /deploy 把项目部署上线
