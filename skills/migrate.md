# 数据迁移

帮用户在不同数据格式之间转换，或从一个系统迁移数据到另一个系统。

## 步骤

### 1. 了解迁移需求

> 你要做什么迁移？比如：
> - 格式转换（CSV → JSON、Excel → CSV、JSON → SQL）
> - 数据库迁移（MySQL → PostgreSQL、本地 → 云端）
> - API 数据导入（从某个 API 拉数据存到本地）
> - 数据清洗（去重、格式统一、缺失值处理）

### 2. 分析源数据

- 读取源数据文件或连接，检查格式和结构
- 统计数据量（行数、字段数）
- 发现潜在问题（编码、缺失值、格式不一致）

### 3. 生成迁移脚本

- **读取源数据** — 处理各种编码和格式
- **数据转换** — 字段映射、类型转换、格式统一
- **数据验证** — 检查转换后的数据是否完整
- **写入目标** — 输出到目标格式或系统
- **错误日志** — 记录跳过或失败的记录

### 4. 执行和验证

先用少量数据测试，确认无误后处理全量数据。

> 迁移完成。共处理 N 条记录，成功 N 条，跳过 N 条。
> 跳过的记录已记录在错误日志中。
