# 性能优化

帮用户找到性能瓶颈并给出具体的改进方案。

## 步骤

### 1. 确定优化目标

> 你想优化什么？
> - 网页加载速度（首屏渲染慢、页面太大）
> - 后端响应时间（API 慢、数据库查询慢）
> - 构建速度（打包太慢）
> - 内存或 CPU 占用

### 2. 分析性能问题

根据优化目标，检查常见瓶颈：

**网页加载：**
- 图片是否压缩、是否使用现代格式（WebP）
- CSS/JS 是否合并、压缩
- 是否有不必要的第三方脚本
- 是否使用了懒加载
- 字体加载是否阻塞渲染

**后端响应：**
- 数据库查询是否有 N+1 问题
- 是否缺少索引
- 是否有不必要的重复计算
- 是否可以加缓存

**构建速度：**
- 是否有不必要的依赖
- 是否可以拆分构建

### 3. 输出优化方案

按影响大小排序，给出具体建议：

```
## 性能分析报告

### 高影响
- [问题] 具体描述 → [方案] 怎么改 → [预期效果] 提升多少

### 中影响
- ...

### 低影响
- ...
```

### 4. 执行优化

用户选择后，逐个执行优化。每个优化后验证效果。

> 优化完成。主要改动：xxx。
